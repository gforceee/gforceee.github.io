Sting、StringBuffer、StringBuilder的比较

发表于 2017-03-04 | 分类于 java

##三者关系简介
针对字符串的处理Java提供了三种类别，分别是String,StringBuffer和StringBuilder。三者有很多相似之处，当然也有也有一些区别，我们可以在不同的场合用不同的类来处理。这三类的大致结构如下：

三类的主要不同点为：

  • String类一旦创建就不能在改变了。
  • StringBuffer和StringBuilder结构基本一样，不同之处在于，StingBuffer线程安全的而StringBuilder线程不安全。
  • 三个类的处理速度：StringBuilder > StingBuffer > string。

String详解

String类是一个非常特殊的类，他可以像基本数据类型一样采用类似于String str=“123”这种方式来赋值，而且字符串存储在一个公共池中。这有利于对具有相同内容的字符串，以节省存储存储的共享。需要注意的是new操作符分配的String对象都存储在堆中，并没有对相同内容的共享存储

###不可变定义：
不可变对象是指对象的状态在被初始化以后，在整个对象的生命周期内，不可改变。

###为什么要不可变：

  1. 安全首要原因是安全，不仅仅体现在你的应用中，而且在JDK中，Java的类装载机制通过传递的参数（通常是类名）加载类，这些类名在类路径下，想象一下，假设String是可变的，一些人通过自定义类装载机制分分钟黑掉应用。如果没有了安全，Java不会走到今天
  2. 性能 string不可变的设计出于性能考虑，当然背后的原理是string pool，当然string pool不可能使string类不可变，不可变的string更好的提高性能。
  3. 线程安全 当多线程访问时，不可变对象是线程安全的，不需要什么高深的逻辑解释，如果对象不可变，线程也不能改变它。

###String类使用”+”来连接字符的整个过程描述：

  1. String str= “a”;创建一个String对象，str 引用到这个对象。
  2. 再创建一个长度为str.length() 的StringBuffer 对象。
  3. StringBuffer strb=new StringBuffer（str）。
  4. 调用StringBuffer的append（）方法将”b“添加进去,strb.append（”b”）。
  5. 调用strb 的toString()方法创建String对象，之前对象失去引用而str重新引用到这个新对 象。
  6. 同样在创建StringBuffer对象 调用append（）方法将”c“添加进去,调用toString() 方法 创建String对象。
  7. 再将strb引用到 新创建的String对象。之前对象失去引用之后存放在常量池中，等待垃圾回收。
